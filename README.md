## 系统概述

本系统是一个**企业级多智能体协同工作平台**，融合了当前业界最先进的多智能体框架的核心特性：

| 参考框架 | 核心创新 | 本系统实现 |
|---------|---------|-----------|
| **LangChain** | Supervisor 分层协调、上下文管理 | ✅ SupervisorAgent 协调层 |
| **AutoGPT** | 循环架构、链式思维、自我反思 | ✅ 反射机制、试错调整 |
| **CrewAI** | 角色专精、任务编排、异步执行 | ✅ 角色定义、并行执行 |
| **AutoGen** | 对话协作、成本监控、错误恢复 | ✅ 实时监控、自动恢复 |

### 设计理念

本系统遵循以下核心设计理念：

1. **分层解耦**：采用 Supervisor 协调层与执行层分离的设计，提高系统可维护性
2. **自我进化**：内置学习机制，从历史任务中自动提取优化策略
3. **成本可控**：实时监控 Token 消耗，支持预算提醒和成本优化
4. **反思机制**：引入 AutoGPT 的反思模式，自动评估结果质量并提出改进建议

## 核心特性

### 1️⃣ Supervisor 协调层

Supervisor Agent 是系统的核心协调器，负责：

- **任务理解**：分析用户意图，判断任务类型（代码生成/分析/研究等）
- **复杂度评估**：自动评估任务复杂度（高/中/低）
- **策略制定**：根据任务类型和复杂度制定最优执行策略
- **Agent 协调**：管理多个执行 Agent 的协作流程

```python
# Supervisor 自动分析任务
understanding = await supervisor.understand_task("创建一个完整的电商系统")
# 返回：任务类型、复杂度评估、Token 估算、执行策略
```

### 2️⃣ 任务智能分解

基于 LangChain 的任务分解模式：

- **递归分解**：复杂任务自动拆分为可管理的子任务
- **依赖分析**：识别任务间的依赖关系
- **并行识别**：标记可并行执行的任务组
- **优先级排序**：根据任务重要性和依赖关系优化执行顺序

### 3️⃣ 自我反思机制

借鉴 AutoGPT 的反思模式：

- **质量评估**：从成功率、输出长度、Token 效率等多维度评估
- **问题识别**：自动识别执行过程中的问题
- **改进建议**：生成针对性的优化建议
- **智能重试**：根据反思结果决定是否需要重新执行

### 4️⃣ 成本实时监控

参考 AutoGen 的成本追踪系统：

- **Token 统计**：实时追踪输入/输出 Token 消耗
- **费用计算**：支持多种模型（GPT-4/3.5/Claude）的定价计算
- **预算提醒**：支持日/月预算设置和阈值提醒
- **成本报告**：生成详细的使用报告和优化建议

### 5️⃣ 知识库与自我学习

系统具备持续学习能力：

- **模式提取**：从成功执行中提取可复用的模式
- **策略优化**：根据历史数据优化执行策略
- **知识积累**：建立任务类型与最优执行方案的映射关系
- **性能追踪**：记录和分析系统性能指标

### 6️⃣ 异步并行执行

基于 CrewAI 的并发执行设计：

- **任务分组**：自动识别可并行的任务组
- **异步执行**：使用 asyncio 实现高效并发
- **结果聚合**：自动整合并行任务的执行结果
- **错误隔离**：单任务失败不影响其他任务

## 详细功能说明

### 1. Supervisor Agent

Supervisor Agent 是系统的"大脑"，负责理解用户意图并制定执行策略。

#### 主要功能

| 功能 | 说明 | 示例 |
|-----|------|-----|
| 任务分类 | 自动识别任务类型 | 代码生成、分析、研究等 |
| 复杂度评估 | 评估任务复杂程度 | 高/中/低三档 |
| Token 估算 | 预估 Token 消耗 | 辅助成本控制 |
| 策略制定 | 生成最优执行策略 | 单Agent/多Agent |

#### 使用示例

```python
from agents.supervisor import SupervisorAgent

supervisor = SupervisorAgent()

# 分析任务
result = await supervisor.understand_task("创建一个完整博客系统")

print(f"任务类型: {result['task_type']}")
print(f"复杂度: {result['complexity']}")
print(f"预计Token: {result['estimated_tokens']}")
print(f"执行策略: {result['strategy']}")
```

### 2. 任务分解器

智能将复杂任务分解为可执行的子任务。

#### 分解策略

- **递归分解**：对于高复杂度任务，支持多层级分解
- **依赖分析**：识别任务间的先后顺序和依赖关系
- **并行识别**：自动标记可并行执行的任务

#### 复杂度判断

| 关键词 | 复杂度 | 示例 |
|--------|--------|------|
| 系统、项目、完整 | 高 | 创建一个完整的电商系统 |
| 模块、功能 | 中 | 实现用户认证模块 |
| 简单、基础 | 低 | 写一个排序函数 |

### 3. 执行器

负责实际执行分解后的子任务。

#### 执行模式

- **顺序执行**：依赖关系强的任务按顺序执行
- **并行执行**：独立任务并发执行，提高效率
- **混合执行**：根据任务特性自动选择最优模式

#### 错误处理

- 自动重试机制（默认 3 次）
- 错误隔离，单任务失败不影响整体
- 详细的错误日志记录

### 4. 反思器

引入 AutoGPT 的反思机制，自动评估执行结果。

#### 评估维度

| 维度 | 指标 | 阈值 |
|------|------|------|
| 成功率 | 子任务成功比例 | ≥70% |
| 输出质量 | 平均输出长度 | ≥50 字符 |
| Token 效率 | Token 利用率 | 越高越好 |

#### 质量等级

- **优秀（Excellent）**：综合评分 ≥80
- **良好（Good）**：综合评分 60-79
- **一般（Fair）**：综合评分 40-59
- **较差（Poor）**：综合评分 <40

### 5. 成本追踪器

实时监控和分析 Token 消耗及费用。

#### 支持的模型

| 模型 | 输入价格（$/1K） | 输出价格（$/1K） |
|------|-----------------|-----------------|
| GPT-4 | 0.03 | 0.06 |
| GPT-3.5-turbo | 0.0005 | 0.0015 |
| Claude-3 | 0.015 | 0.075 |
| 默认 | 0.01 | 0.03 |

#### 预算管理

```python
from core.cost_tracker import CostTracker

# 设置预算

print(f"剩余预算: ${stats['budget_remaining']:.2f}")
```

### 6. 学习器

系统具备自我学习和进化能力。

#### 学习机制

- **模式提取**：从成功执行中提取可复用的模式
- **策略优化**：根据历史数据优化执行策略
- **知识积累**：建立任务类型与最优执行方案的映射

#### 2. 事件驱动

使用 asyncio 实现事件驱动的任务调度：

```python
async def execute_with_monitoring(self, subtasks):
    # 并行执行独立任务
    parallel_tasks = [t for t in subtasks if t['parallel']]
    
    if parallel_tasks:
        results = await asyncio.gather(*[
            self.executor.execute(task) 
            for task in parallel_tasks
        ])
    
    return results
```

#### 3. 策略模式

根据任务类型选择不同的执行策略：

```python
strategy = {
    ('code_generation', 'high'): {
        'mode': 'multi_agent',
        'agents': ['decomposer', 'executor', 'optimizer'],
        'parallel': True
    },
    ('analysis', 'low'): {
        'mode': 'single_agent',
        'agents': ['executor'],
        'parallel': False
    }
}
```

## 常见问题

### Q1: 如何处理 API 调用失败？

**A：** 系统内置自动重试机制，默认重试 3 次。每次重试会记录日志，方便排查问题。

```python
config = {
    'max_retries': 3,
    'retry_delay': 1.0,  # 重试间隔（秒）
    'timeout': 120        # 超时时间（秒）
}
```

### Q2: 如何控制成本？

**A：** 使用成本追踪器设置预算提醒。

```python
tracker = CostTracker()
tracker.budget['daily_limit'] = 10.0
tracker.budget['alert_threshold'] = 0.8
```

### Q3: 知识库数据存储在哪里？

**A：** 默认存储在 `./data/knowledge_base/` 目录，包括：

- `patterns.json`：学习到的模式
- `workflows.json`：工作流配置
- `metrics.json`：性能指标

### Q4: 如何扩展新的 Agent？

**A：** 遵循以下步骤：

1. 在 `agents/` 目录创建新的 Agent 类
2. 继承基础 Agent 接口
3. 实现 `execute()` 方法
4. 在 `main.py` 中注册新 Agent

### Q5: 系统支持多语言吗？

**A：** 支持。系统内置中文关键词识别，同时也支持英文任务的处理。
